# Pacing Analyzer Skill

## Overview
Skill สำหรับวิเคราะห์และปรับจังหวะเรื่อง ไม่ให้เร็วเกินไปหรือช้าเกินไป

---

## ความสามารถหลัก

### 1. Chapter Pacing Analysis (วิเคราะห์จังหวะบท)
- ตรวจสอบความเร็ว-ช้าของแต่ละบท
- หา "ตอนน้ำ" ที่ไม่จำเป็น
- ระบุจุดที่รีบเกินไป

### 2. Story Arc Pacing (จังหวะโครงเรื่อง)
- วิเคราะห์ภาพรวมทั้งเรื่อง
- ตรวจสอบ Rising/Falling Action
- หา Sagging Middle

### 3. Scene Transition (การเชื่อมฉาก)
- วิเคราะห์การเปลี่ยนฉาก
- หาจุดที่กระโดดเกินไป
- หาจุดที่ลากยาวเกินไป

---

## ปัญหา Pacing ที่พบบ่อย

### 1. Sagging Middle (ตอนกลางหย่อน)
```
ปัญหา: ตอนกลางเรื่องไม่มีอะไรเกิดขึ้น น่าเบื่อ

สาเหตุ:
- ตั้ง conflict แล้วปล่อย
- ไม่มี subplot ช่วย
- ตัวละครไม่พัฒนา

แก้ไข:
- เพิ่ม mini-conflict
- เพิ่ม twist/revelation
- เร่ง subplot
```

### 2. Rushed Ending (จบเร็วเกินไป)
```
ปัญหา: build up มานาน แต่จบไวมาก

สาเหตุ:
- เขียนยาวมาแล้ว อยากจบ
- คิดว่าผู้อ่านรู้แล้ว

แก้ไข:
- ให้เวลา climax
- ให้ resolution ที่เหมาะสม
- อย่าลัด subplot
```

### 3. Slow Start (เริ่มช้าเกินไป)
```
ปัญหา: ใช้เวลา setup นานเกินไป

สาเหตุ:
- อยาก world-build มากเกินไป
- อยากแนะนำตัวละครทุกตัว
- ไม่กล้าเริ่ม conflict

แก้ไข:
- เริ่มที่ action
- World-build ระหว่างทาง
- ให้ conflict ตั้งแต่ต้น
```

### 4. Filler Chapters (ตอนน้ำ)
```
ปัญหา: ตอนที่ไม่ได้เพิ่มอะไรให้เรื่อง

สังเกตได้จาก:
- ลบออกแล้วเรื่องยังสมบูรณ์
- ไม่มี conflict
- ตัวละครไม่เปลี่ยน
- ไม่มีข้อมูลใหม่

แก้ไข:
- ตัดออก หรือ
- เพิ่ม conflict/revelation
```

## Framework การวิเคราะห์ Pacing

### Pacing Score Card

```
┌─────────────────────────────────────────────────────────────┐
│  Chapter/Scene Analysis                                     │
├─────────────────────────────────────────────────────────────┤
│  □ มี Conflict? (ไม่มี = ช้า)                              │
│  □ มี Forward Movement? (เรื่องดำเนินไป?)                   │
│  □ มี Character Development?                               │
│  □ มี Information/Revelation?                              │
│  □ มี Emotional Impact?                                    │
│  □ จบด้วย Hook/Cliffhanger?                               │
├─────────────────────────────────────────────────────────────┤
│  Score: ___/6                                              │
│  0-2 = ช้ามาก (ตัดหรือรวมกับบทอื่น)                        │
│  3-4 = ปานกลาง (ปรับปรุง)                                  │
│  5-6 = ดี (เก็บไว้)                                        │
└─────────────────────────────────────────────────────────────┘
```

### Story Arc Pacing Map

```
Ideal Pacing:

    ↑                    ★ Climax
    │                   /│\
    │                  / │ \
    │       Rising    /  │  \  Falling
    │       Action   /   │   \ Action
    │              /     │    \
    │   Setup    /       │     \ Resolution
    │           /        │      \
    │──────────/─────────│───────\──────→
    │                    │
    Chapter 1     Middle      End

❌ Bad: Flat line (ไม่มีขึ้นลง)
❌ Bad: Climax เร็วเกินไป
❌ Bad: Falling action นานเกินไป
```

## เทคนิคปรับ Pacing

### 1. เร่งจังหวะ (Speed Up)

```
- ตัดบรรยายยาวๆ
- ใช้ประโยคสั้นลง
- เพิ่ม dialogue
- กระโดดข้ามเวลา
- เพิ่ม action

ก่อน:
"เธอตื่นขึ้นมา มองนาฬิกา 7 โมงเช้า
เธอลุกขึ้นไปอาบน้ำ แปรงฟัน แต่งตัว
กินข้าวเช้า ดื่มกาแฟ อ่านข่าว
แล้วก็ออกจากบ้านไปทำงาน..."

หลัง:
"7 โมงเช้า
ถึงเวลาเผชิญหน้ากับเขาอีกครั้ง"
```

### 2. ชะลอจังหวะ (Slow Down)

```
- เพิ่มรายละเอียด sensory
- เพิ่ม internal monologue
- บรรยายอารมณ์
- ให้เวลา moment สำคัญ

ก่อน:
"เขาบอกรักเธอ เธอยิ้ม"

หลัง:
"เขาบอกรักเธอ

คำสามคำที่เธอรอมานาน
ดังก้องอยู่ในหู

เธอมองเข้าไปในตาเขา
หาความจริงในนั้น
และพบแต่ความรัก

ริมฝีปากเธอยกขึ้น
รอยยิ้มที่มาจากหัวใจ"
```

## ตัวอย่างการวิเคราะห์

### Input: โครงเรื่อง 30 ตอน

```
ตอน 1-5: แนะนำตัวละคร, โลก, ปัญหา
ตอน 6-10: ตัวเอกเริ่มเดินทาง
ตอน 11-20: เดินทางต่อ, พบเพื่อน, ฝึกฝน
ตอน 21-25: เข้าใกล้เป้าหมาย
ตอน 26-28: เผชิญหน้าศัตรู
ตอน 29-30: จบ
```

### Output: Pacing Analysis

```
⚠️ ปัญหาที่พบ:

1. Slow Start (ตอน 1-5 = 17%)
   - 5 ตอนสำหรับ setup มากเกินไป
   - แนะนำ: รวมเป็น 2-3 ตอน

2. Sagging Middle (ตอน 11-20 = 33%)
   - "เดินทางต่อ" ไม่ specific
   - "ฝึกฝน" ไม่มี conflict
   - แนะนำ: เพิ่ม mini-villain, subplot

3. Rushed Ending (ตอน 26-30 = 17%)
   - Climax + Resolution แค่ 5 ตอน
   - แนะนำ: ขยายให้ 7-8 ตอน

📊 Recommended Restructure:
- Act 1 (Setup): 3 ตอน (10%)
- Act 2 (Conflict): 20 ตอน (67%)
- Act 3 (Resolution): 7 ตอน (23%)
```

## Checklist

- [ ] ทุกบทมี purpose
- [ ] ไม่มี "ตอนน้ำ"
- [ ] Setup ไม่นานเกินไป
- [ ] ตอนกลางไม่หย่อน
- [ ] Climax ได้เวลาเพียงพอ
- [ ] Resolution สมเหตุสมผล
- [ ] การเปลี่ยนฉากลื่นไหล

---

## Tips

### DO:
- วิเคราะห์ก่อนเริ่มเขียน
- ทำ Pacing Map ทั้งเรื่อง
- ตรวจสอบทุกบทว่ามี purpose
- ยอมตัดถ้าไม่จำเป็น

### DON'T:
- ยึดติดกับจำนวนบท
- เขียน filler เพื่อให้ครบ
- รีบจบเพราะเบื่อ
- ลืม subplot
