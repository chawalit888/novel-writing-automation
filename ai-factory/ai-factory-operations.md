# AI Novel Factory - คู่มือปฏิบัติการ

## หลักการหลัก

คุณคือเจ้าของค่ายนิยาย ใช้ AI เป็นเครื่องมือผลิต ทุกเรื่องผ่านการวิจัยมาก่อนแต่ง ใช้ปริมาณหา hit แล้วทุ่มเวลาเฉพาะเรื่องที่ทำเงิน

**เป้าหมาย:** 10-15 เรื่อง active ทำเงินพร้อมกัน → 500,000-1,000,000 บาท/เดือน

---

## 1. กระบวนการผลิต

### ระบบที่ใช้: Claude Code เท่านั้น

| ขั้นตอน | Skill/เครื่องมือ |
|---------|-----------------|
| วิจัยแนว/ตลาด | Knowledge Base + Brave Search |
| สร้างตัวละคร | `character-creator` + `character-chemistry-builder` |
| วาง plot | `plot-manager` |
| สร้างโลก | `world-building` |
| เขียนตอน | `novel-writer` + `dialogue-master` + `tension-building` |
| สร้าง hook/cliffhanger | `hook-cliffhanger-specialist` |
| เขียนฉาก NC | `nc-automation` (Ollama local) + `nc-scene-polish` + `nc-translation-specialist` |
| QC ก่อนลง | QC Pipeline 6 ขั้นตอน (ดู `qc-pipeline.md`) |
| ปก | Cover Design Team |
| การตลาด | Marketing Team |

### มาตรฐานการผลิต
- ความยาวต่อตอน: 4,800-6,000 ตัวอักษร (ไม่น้อยกว่า 4,800)
- ไฟล์: `.txt` UTF-8
- ตั้งชื่อ: `ตอนที่XX-[ชื่อตอน].txt`
- ทุกตอนต้องจบด้วย cliffhanger
- ห้ามมี filler (ตอนที่ตัดออกแล้วเรื่องไม่เปลี่ยน)

## 2. สูตรปล่อยเรื่องใหม่

### การเปิดเรื่อง (10-Chapter Drop)

| วัน | จำนวนตอน | หมายเหตุ |
|-----|---------|---------|
| Day 0 (วันเปิด) | 10 ตอนรวด | ตอน 1-10 ฟรีทั้งหมด |
| Day 1 | +2 ตอน | ตอน 11-12 เริ่มคิดเงิน |
| Day 2 | +1-2 ตอน | |
| Day 3+ | วันละ 1 ตอน | ต่อเนื่องทุกวัน |

### กฎการเปิดเรื่อง
- ปล่อยสัปดาห์ละ 2-3 เรื่องใหม่
- ห้ามเปิด 2 เรื่องวันเดียวกัน
- เว้นอย่างน้อย 2 วันระหว่างเปิดเรื่อง
- ตัวอย่าง: จันทร์เปิดเรื่อง A, พฤหัสเปิดเรื่อง B, เสาร์เปิดเรื่อง C

### ถ้ามี stock เรื่องจบแล้วพร้อมปล่อย
- ห้ามลงพร้อมกัน ทยอยสัปดาห์ละ 2 เรื่อง
- อย่าปล่อยครบทีเดียว (คน binge ฟรี ไม่เติม coin)
- ปล่อยฟรี 10 ตอน → เปิดขาย → ลงวันละ 1 ตอน
- = มีรายได้ยาว 3-4 เดือน/เรื่อง

## 3. Paywall Strategy

### โครงสร้างราคา

| ตอน | ราคา | หมายเหตุ |
|-----|------|---------|
| ตอน 1-10 | ฟรี | ดึงคนอ่านให้ติด |
| ตอน 11+ | คิดเงิน | อาจมีบางตอนฟรีแทรก (กระตุ้นให้กลับมาอ่าน) |
| ตอน NC ทุกตอน | คิดเงินเสมอ | ไม่มียกเว้น NC ต้องจ่ายทุกตอน |

### Paywall Hook (ตอนที่ 10)
- ตอนที่ 10 ต้องเป็น cliffhanger แรงที่สุดของเรื่อง
- สูตร: [Revelation] + [Danger/Decision] + [Emotional Peak] + [Unanswered Question]
- ใช้ `hook-cliffhanger-specialist` skill ที่ความแรง 5/5 ดาว
- คนอ่านต้องรู้สึก "ต้องรู้ว่าเกิดอะไรขึ้น" → ยอมจ่ายเงิน

### NC เป็นตัวทำเงิน
- NC scene แรก: ตอน 12-15 (หลัง paywall ไม่นาน → คนเพิ่งจ่ายเงิน ได้คุ้ม)
- NC loop: ทุก 3-5 ตอน (NC → เกือบ NC → tension → หึงหวง → NC)
- NC คิดเงินทุกตอน → เป็น revenue driver หลัก

---

## 4. ดูผลหลังลง (Test Period)

### วิธีการ
1. ลง 15-20 ตอนแรก (10 ฟรี + 5-10 เสียเงิน)
2. สังเกต 14-21 วัน
3. ดูตัวเลขจริงจากแพลตฟอร์ม

### สิ่งที่ดู (ยังไม่ตั้ง KPI ตัวเลข - รอข้อมูลจริงจากเรื่องที่ลงแล้ว)
- คนอ่านเยอะขึ้นหรือลดลง?
- มีคนติดตามไหม?
- มีคอมเมนต์ไหม? (โดยเฉพาะ "อยากอ่านต่อ", "พระเอกโหดจัง")
- มีคนเติม coin ซื้อตอนต่อไหม?

### การตัดสินใจ

**ตัวเลขดี (คนอ่าน/ติดตาม/เติม coin):**
- แต่งต่อยาว → เป้า 120-200 ตอน
- เร่งผลิตวันละ 1 ตอน
- เปิด NC pipeline
- ดันการตลาด (teaser, LINE broadcast, social)
- สร้างปก (cover-design-team)

**ตัวเลขไม่ดี (เงียบ ไม่มีคนอ่าน/ติดตาม):**
- พักเรื่อง หรือเขียนจบสั้นๆ 2-3 ตอน
- ปล่อยตอนที่เหลือฟรี (สร้าง goodwill กับผู้อ่าน)
- บันทึกบทเรียน: ทำไมเรื่องนี้ไม่ขึ้น?
- ไม่ต้องเสียใจ → เอาเวลาไปแต่งเรื่องใหม่ที่มีโอกาสมากกว่า

**ตัวเลขกลางๆ (มีคนอ่านบ้าง แต่ไม่โดดเด่น):**
- แต่งต่ออีก 10-15 ตอน ดูแนวโน้ม
- ถ้าเพิ่มขึ้น → ดัน
- ถ้าเท่าเดิม/ลด → พัก

## 5. Scale เรื่องที่ Hit

### แผนแต่งยาว 120-200 ตอน

**โครงสร้างเรื่องยาว:**
- ตอน 1-10: ฟรี → สร้างความสนใจ + แนะนำตัวละคร
- ตอน 11-30: เสียเงิน → conflict หลัก + NC เริ่มเข้ม
- ตอน 31-80: เสียเงิน → plot twist + sub-plot + NC loop ทุก 3-5 ตอน
- ตอน 81-120: เสียเงิน → climax + NC เข้มข้นขึ้น
- ตอน 121-200: เสียเงิน → resolution + epilogue + bonus NC

**กฎการแต่งยาว:**
- NC loop ทุก 3-5 ตอน (สลับ: NC เต็ม → เกือบ NC → tension → หึงหวง → possessive)
- Cliffhanger ทุกตอน ตัดจบแบบค้าง
- Plot twist ทุก 15-20 ตอน (ไม่ให้เรื่องจืด)
- ตัวละครใหม่/sub-plot เข้าทุก 25-30 ตอน (สร้างความสดใหม่)
- ห้ามมี filler → ทุกตอนต้องขยับ plot หรือพัฒนาตัวละคร

**การผลิตเรื่อง hit:**
- ใช้ Claude Code เท่านั้น (คุณภาพสูง)
- QC score >= 85/100
- ต้อง buffer ล่วงหน้า 10+ ตอน (อย่าให้ stock หมด)
- วันละ 1 ตอน = 120 ตอนใน 4 เดือน

## 6. KPI (ยังไม่กำหนดตัวเลข)

**แผน:** รอลงเรื่องจริง 3-5 เรื่อง แล้วเก็บข้อมูลจาก Tunwalai
- วิวเฉลี่ยต่อตอน
- จำนวนผู้ติดตามต่อเรื่อง
- คอมเมนต์ต่อตอน
- Coin revenue ต่อตอน

เอาตัวเลขจริงมาตั้งเป็น baseline แล้วค่อยเขียน KPI Framework ตามข้อมูลจริง
